Digital Omotenashi
Three Pillars
Japanese hospitality principles, operationalised into AI
P

Personalised (AIGD)

Content is dynamically generated from this specific user's profile, behavioural patterns, and daily inputs. No two sessions are alike. "Know the user first, guide them better."

A

Anticipatory

Petch does not wait for the user to ask. It proactively prompts reflection, surfaces unrecognised needs, and identifies symptoms the user may not connect to a broader condition.

E

Empathetic

Human-centred, supportive language throughout. The Pet Companion reacts to engagement, not health scores — the user feels understood, never judged, never overwhelmed.

Phase 3 — Action · Closes knowing-doing gap · Survey's #1 feature (69%)

Doing

Committed Action · Immediate, Personalised Next Steps
User commits to 1 concrete action before session closes
Committed action auto-fills Mission Slot 1 in the Actions tab
Petch proactively suggests Slots 2 + 3, targeting user's weakest areas
Completing missions earns coins → closes the gamification loop (86% survey preference)
P

Slot 1 = user's own words. Slots 2 + 3 target their specific weak patterns from history — directly validates survey's #1 feature: "immediate, applicable guidance". Personalisation closes the knowing-doing gap.

A

Actions tab pre-populated before the user opens it — Petch assigns without being asked, at session close. Externalises the procrastination barrier by removing friction.

E

Max 3 missions prevents overwhelm (addresses 17% consistency barrier). Framed as commitment not instruction. Skipping allowed without penalty — the user chose it. Pet celebrates wins, never punishes.
